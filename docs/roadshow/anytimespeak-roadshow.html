AnytimeSpeak

AI 英语口语陪练系统

让英语学习者在真实场景中敢开口、能纠错、可复盘。

汇报人：覃迦迎 联系方式：24213020053@m.fudan.edu.cn
项目背景 / 痛点

很多学习者不是不会英语，而是缺少稳定开口的环境。

02

口语学习的难点，不只是“说对”，更是能在低压力环境里反复开口，并获得及时反馈。

不敢开口

缺少低压力、可反复练习的口语环境

缺少反馈

说完后不知道哪里不自然、该怎么改

难以复盘

练习结束后没有沉淀，看不到持续进步

AnytimeSpeak 希望把一次开口，变成一次有反馈、可复盘的口语练习。

产品闭环

把一次对话，组织成完整的口语练习链路。

03

用户从场景进入练习，通过多轮对话获得即时反馈、发音测评、课后总结与历史复盘。

场景选择输入AI 对话即时反馈发音测评总结历史复盘

核心不是“聊几句”，而是让每次练习都留下可改进的依据。

真实练习页：对话、输入和反馈集中在同一学习工作台。

核心功能

核心功能已经形成可演示的产品路径。

04

场景化练习

面试、点餐、会议、旅行、日常交流。

语音 / 文本输入

麦克风输入可用，文本输入作为稳定 fallback。

AI 多轮对话

围绕角色、场景和目标持续追问。

即时反馈

给出表达建议、原因说明和可替代表达。

发音测评

展示总分、流利度、准确度、完整度。

总结复盘

沉淀表现、问题和下一步建议。

截图展示的是已实现功能，而不是方案示意。

场景设计

不是自由聊天，而是有目标、有角色、有反馈重点的练习。

05

为什么要先选场景

  • 限制话题范围，避免练习发散。
  • 明确 AI 与用户角色，降低开口压力。
  • 提前设定反馈重点，便于总结复盘。

场景化让 AI 陪练从“随便聊”变成“围绕目标练”。

系统架构

前端、后端、AI / 语音能力和数据存储分层协作。

06

前端

React + Vite + TypeScript，承载场景选择、练习页、反馈、总结和历史记录。

后端

FastAPI 编排 session、chat、feedback、summary、pronunciation、history。

AI / 语音能力

LLM、ASR、发音测评 provider，并保留 mock fallback。

SQLite

保存 sessions、messages、feedback、summary 和 practice history。

架构目标是先保证完整链路稳定可演示，再逐步替换更强的 provider 和存储方案。

Session-based Flow

让 AI 记住这次练习，而不是每轮重新开始。

07

没有 session

每轮对话割裂，场景、角色、目标和历史表现都难以沉淀。

使用 session

绑定 scenario、story seed、messages、feedback 和 summary，练习过程可追踪。

Session 是反馈、总结和历史复盘能够成立的基础。

Provider + Fallback
